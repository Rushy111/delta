About Umbrafiles

What is Umbrafiles?

Umbrafiles is a private, peer-to-peer workspace designed for seamless device connectivity. It enables direct file transfers, screen mirroring, and webcam streaming between your devices — all without relying on any third-party servers or cloud infrastructure.

The name "Umbra" reflects the shadow-like privacy of the system: your data exists only between your devices, invisible to the outside world.

Core Features

  • File Transfer — Send any file type directly between devices using WebRTC DataChannels. No size limits imposed by servers.
  • Screen Mirroring — Stream your phone's screen to your PC or vice versa in real-time with minimal latency.
  • Phone as Webcam — Use your phone's camera system as a high-quality webcam for any video application on your PC.
  • Zero-Knowledge Architecture — No data is stored on any server. Everything flows directly between your devices.

Technology Stack

  • WebRTC — For peer-to-peer data and media streaming
  • JavaScript (ES6+) — Client-side application logic
  • HTML5 / CSS3 — Structure and styling
  • Manual Signaling — SDP exchange via copy-paste for maximum privacy

Development Roadmap

  • 2026 Q1 — Initial release (this version): Core P2P engine, file transfer, basic screen mirroring
  • 2026 Q2 — Enhanced screen mirroring with adaptive bitrate, webcam with filters
  • 2026 Q3 — Built-in signaling server (optional), multi-device support
  • 2026 Q4 — Public beta release (if deemed ready), community features

Security Information

Encryption

Umbrafiles uses military-grade encryption for all data in transit:

  • DTLS-SRTP — Encrypts all media streams (screen sharing, webcam)
  • SCTP over DTLS — Encrypts all file transfers via DataChannels
  • ECDH Key Exchange — Establishes unique session keys for each connection

Zero-Knowledge Architecture

Umbrafiles operates on a zero-knowledge model:

  • No central server exists to compromise
  • No data passes through any intermediary
  • The creator cannot access your data even if compelled
  • All encryption/decryption happens on your devices

Connection Security

  • Manual Signaling — SDP exchange is done manually, preventing man-in-the-middle attacks
  • Device Verification — Each connection is unique and tied to specific devices
  • Session Expiry — Connections can be terminated at any time by either party

Password Protection

The application is protected by a password that:

  • Is verified locally (never sent over network)
  • Uses hashing for verification
  • Has rate limiting to prevent brute-force attacks
  • Locks out after repeated failed attempts

Best Practices

  • Only connect to devices you physically own and trust
  • Never share your access password
  • Never share signaling data (SDP offers/answers) with others
  • Keep your devices and browsers updated
  • Terminate connections when not in use
  • Use a strong, unique password

Connect Devices

How to Connect Your Devices

Umbrafiles uses WebRTC for direct peer-to-peer connections. Since this is a static website with no signaling server, you'll manually exchange connection data.

Step-by-Step Guide

  1. On your PC: Open Umbrafiles and go to any feature page (Transfer, Mirror, or Webcam)
  2. Click "Generate Offer" — This creates a connection invitation
  3. Copy the offer and send it to your phone (via any method: email, messaging app, etc.)
  4. On your phone: Open Umbrafiles in your browser, paste the offer, and click "Generate Answer"
  5. Copy the answer and send it back to your PC
  6. On your PC: Paste the answer and click "Accept Answer"
  7. Connected! You can now use the selected feature
ℹ️ Tip: Each feature (Transfer, Mirror, Webcam) has its own independent connection. You can run multiple simultaneous connections if needed.

Requirements

  • Both devices must have a modern browser (Chrome, Firefox, Edge, Safari)
  • Both devices need internet access (for WebRTC negotiation)
  • Camera/screen permissions must be granted
  • Firewalls must allow WebRTC traffic (usually UDP ports 10000-60000)
